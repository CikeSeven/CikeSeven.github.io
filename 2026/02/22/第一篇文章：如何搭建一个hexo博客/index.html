第一篇文章：如何从零搭建一个 Hexo 博客

1.6k

偶然逛到别人的博客，看了几篇文章觉得挺有意思，突然萌生了一个念头：要是自己也有这样一个网站，能记录下我的日常和经历就好了。于是，忙活了一下午，这个小站就这么诞生了。

本站是使用 Hexo 搭建的，选择它的最大原因就是——轻量且白嫖。不需要购买昂贵的服务器，直接将生成的静态页面托管到 GitHub Pages 上，基本可以说是零成本（当然也可以买一个低价域名）。

如果你也想零成本搭建一个同款博客，那就继续往下看吧。

绑定专属域名（可选）

如果你不想使用 GitHub 赠送的二级域名，刚好自己手里有一个域名，还可以将其绑定过来。

  1. 配置 DNS 解析： 在你的域名管理后台，添加一条新的记录：
    • 类型选择：CNAME
    • 主机记录：推荐blog
    • 记录值：填 你的用户名.github.io
  2. 配置 Hexo 域名文件： 打开本地博客的 source 文件夹，新建一个名为 CNAME 的文件（注意全大写且没有任何后缀）。用记事本打开它，写上你的专属域名（例如 domain.xyz），保存并关闭。
  3. 配置 GitHub 仓库： 在 GitHub 打开你的博客仓库，点击顶部的 Settings -> 左侧的 Pages。在 Custom domain 框内填写你的域名并点击 Save。
  4. 重新部署： 再次运行 hexo cl && hexo g && hexo d。

等几分钟 DNS 刷新后，应该能通过域名访问了。

主题与插件配置

目前的默认主题比较单调，但 Hexo 社区的繁荣在于它极具可玩性的主题和插件。

  • 挑选主题： 可以看看 Hexo 官方主题库 ，也可以在 GitHub 搜索 hexo-theme 寻找高星项目。本站目前使用的是现代且功能强大的卡片流主题：Solitude。
  • 安装插件： Hexo 支持字数统计、RSS、站点地图等各种插件。大多数优秀的主题文档都会提供推荐配合使用的插件清单，跟着文档使用 npm install 插件名 安装即可。

附录：常用的 Hexo 备忘命令

俗话说好记性不如烂笔头，把常用的指令列在这里，等以后忘了方便自己回来翻查：

命令 简写 说明
hexo init [文件夹] 无 在指定文件夹新建一个 Hexo 博客
hexo clean hexo cl 清除已生成的静态文件和缓存
hexo generate hexo g 将 Markdown 渲染生成静态网页
hexo server hexo s 启动本地预览服务器 (localhost:4000)
hexo new page "页面名" hexo n page 新建一个独立页面 (如关于我、留言板)
hexo new post "标题" hexo n 新建一篇常规文章
hexo deploy hexo d 部署生成的网页到云端 (如 GitHub)

我自己就用得上这么些，更多指令的话可以查阅 Hexo 官方文档
